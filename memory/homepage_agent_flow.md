# 홈페이지 팀에이전트 워크플로우

## 프로젝트: KACA 홈페이지
- 상세: `memory/kaca_homepage_project.md`

## 작업 프로세스 (필수 순서)

### 1단계: 요청 수신 + Discord 응답
- 대표님 요청 확인
- Discord 즉시 응답 (작업 계획/확인)

### 2단계: 구현
- 코드 작성/수정
- 로컬 빌드 확인 (`next build` 성공)

### 3단계: 배포
- deploy 폴더에 파일 복사
- Vercel 프로덕션 배포

### 4단계: 최종 검증 (절대 생략 금지)
**배포 후 반드시 Playwright MCP로 실제 배포 URL에서 검증**

#### 4-1. 기능 검증 (evaluate)
```
모든 페이지별:
- URL 정상 이동 여부
- Nav 링크 클릭 → 페이지 전환 확인
- 이미지 로딩 상태 (complete && naturalWidth > 0)
- 주요 콘텐츠 존재 확인 (h1, 카드 수, 폼 필드 등)
```

#### 4-2. 레이아웃 검증 (evaluate + screenshot)
```
데스크톱(1440px):
- 요소 위치 좌표 확인 (getBoundingClientRect)
- 그리드/플렉스 배치 정상 여부 (예: 임원진 3/2 배치)
- CSS computed style 확인 (display, flexWrap 등)

모바일(375px):
- 반응형 미디어쿼리 적용 확인
- Nav 숨김 + 햄버거 표시
- 1열/2열 레이아웃 전환
```

#### 4-3. 시각적 검증 (screenshot)
```
각 페이지 fullPage 스크린샷 촬영 → Read로 직접 확인
- 레이아웃 깨짐 없는지
- 텍스트 잘림/오버플로우 없는지
- 이미지 표시 정상인지
- 색상/간격 정상인지
```

#### 4-4. 문제 발견 시
```
문제 발견 → 즉시 수정 → 재빌드 → 재배포 → 재검증
재검증 통과할 때까지 반복
보고는 검증 통과 후에만
```

### 5단계: 검증 완료 보고
- 검증 항목별 결과 포함하여 Discord 보고
- "데스크톱/모바일 검증 완료" 명시
- 구체적 수치 포함 (이미지 N개 로딩OK, 페이지 N개 이동OK 등)

## 절대 금지 사항
- 검증 없이 "완료" 보고
- 스크린샷만 찍고 내용 미확인
- 빌드 성공만으로 보고 (배포 URL에서 실제 확인 필수)
- 대표님께 "확인해주세요" (검증 떠넘기기)
